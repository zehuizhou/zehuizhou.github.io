shell基本命令

发表于 2016-08-18 | 分类于 shell | 阅读次数

学习shell最好自己有个linux系统，可以在虚拟机上面安装个linux系统，然后用xshell连接上，这样会和工作的环境类似，命令大家都要动手敲敲，不然很容易忘，尤其是一些参数，shell的学习不是一朝一夕的，积累是关键2333。

文件相关

  • ls
    • 全称：list 清单 主要功能就是列举出目录下面的所有文件和目录，但是不递归
    • -a 列出隐藏项
    • -l 列出详细信息 有时可以使用ll
    • -1 一行列一个
    • -R 递归列出目录和子目录下的所有文件
  • cp
    • 全称：copy 复制 主要功能就是复制一个文件或者目录
    • -f 强行复制，即使存在也会覆盖
    • -p 保留源文件所有属性
    • -r 递归复制，复制目录时有用
    • 格式：cp source dest
  • mv
    • 全称：move 移动 将一个文件或者是目录移动到另一处
    • 格式：mv source dest
    • 特殊用法，重命名： 同一个目录下文件执行mv a b 将a重命名为b
  • rm
    • remove files or directories
    • 删除文件或者是目录
    • -f 强制删除不用确认
    • tips：有些系统使用了alias（别名）rm = rm –i 所以即使加了-f也需要确认
    • -i 删除前确认
    • -d 删除空目录
    • -r 递归删除目录和，目录下面的内容
    • 格式:
    • rm a
    • rm –rf dir_name
  • chmod
    • change file mod bits更改文件权限
    • 文件权限 读r，写w，执行x,一个rwx为一组
    • 格式：chmod 777 file/dir
      chmod a+x file (a :all u:owner g:group o:other)
  • cat
    • 全称：Concatenate 连接 “猫一下”
    • -n 显示行号
    • -A 显示所有的隐藏字符
    • 格式：
      • cat file1 仅连接file1 也就是显示出file1的内容
      • cat file1 file2 连接file1和file2
    • 那么有一个很好玩的命令叫tac，效果是从倒数第一行开始读
  • touch
    • 就是字面上的理解：“碰”一下
    • 如果没“碰”到就自己创建该文件
  • less more
    • 这两个都是用来查看文件的，对于比较长的文件来说，使用cat会超过一屏的空间，这样会显得不友好，使用less和more就是为了解决这个问题
    • 当然less和more也有区别，less可以往上翻，more不可以
    • 格式：
      • less file
      • more file

目录相关

  • pwd
    • print working directory打印当前目录
  • cd
    • change directory 更改当前工作目录
    • 格式：
      • cd / 切换到根目录
      • cd .. 切换到父目录
      • cd ~ 切换到当前用户的home目录
      • cd – 切换到之前的目录
  • mkdir
    • make directories
    • -p 父目录不存在创建父目录
    • 格式：
      • mkdir aaa
      • mkdir –p aaa/bbb/ccc
  • rmdir
    • remove empty directories只适用于删除空目录
    • -p 删除当前目录之后如果父目录为空则删除父目录
    • 格式：
      • rmdir aaa
      • rmdir –p aaa/bbb/ccc

挂载相关

  • mount
    • mount a file system
    • 挂载一个文件系统
    • -t 指定文件系统的格式
  • umount
    • umount file systems
    • -l lazy模式，在显示device is busy的时候用,在设备不忙的时候卸载设备
      • 格式：
      • umount /mnt
      • umount –l /mnt

压缩相关

  • tar
    • Tape Archive 磁带归档
    • 主要用于文件的压缩和解压
    • -c 打包一个文件
    • -x 解包一个压缩文件
    • -f 后面直接加文件名，为压缩后的文件名
    • -z 采用gzip压缩和解压缩
    • -v 显示整个过程
    • -C 指定解压目录
    • 格式：
      • 压缩命令： tar –zvcf aaa.tar.gz dir_or_file
      • 解压缩: tar –zvxf aaa.tar.gz –C target_dir

硬盘相关

  • fdisk
    • manipulate disk partition table
    • 操作硬盘的分区表
    • fdisk 不能直接使用，后面必须加参数
    • -l
    • 格式：
      • fdisk -l
  • df
    • report file system disk space usage
    • 主要是用来查看目前系统下面的挂载情况和文件系统的剩余空间
    • -h 以人类可读性较好的形式显示 说白了就是讲byte换算成k m g t
    • 格式：
    • df –h

网络相关

  • ifconfig
    • configure a network interface
    • 显示和配置当前的网络接口
    • -a 显示所有的接口，不管接口的连接状态
    • 格式：
      • ifconfig –a
  • ping
    • 发送ICMP报文包
    • 最简单的用途就是用来判断两个机器是否连通（windows注意关闭防火墙）
    • -c 指定次数
    • ping 192.168.20.100 –c 5

帮助相关

  • man、info
    • 查看和命令相关的信息，具体说明都是英文的
      • 格式
      • man man
      • man info
      • info man
      • info info

日志相关

  • dmesg
    • 查看上一次的开机启动信息
    • -L 给重要的信息标色
    • 一般用在压力测试中，根据关键字来判断是否有系统报错
    • 格式：
      • dmesg
      • dmesg –L
      • 对应的文件存储在/var/log 有dmesg和dmesg.old两个文件

进程相关

  • ps
    • processes snapshot 进程快照
    • -e 显示所有的进程
    • -f 显示全格式（包括UID，PPID，和详细的cmd命令）
    • -a 显示终端上的所有进程，包括其他用户的进程
    • -u 指定用户的进程
    • u 详细显示
    • x 显示没有终端的进程
    • 格式：
      • ps –aux
      • ps -ef
  • kill
    • Terminate a process “杀”掉某个进程
    • pid 直接杀掉进程号为pid进程
    • -9 强制杀！
    • 格式：
      • kill pid
      • kill -9 pid
      • 扩展：
      • killall 可以直接指定进程名字而不是进程号，如：killall java

资源相关

  • free
    • 查看系统下面的内存基本信息
    • -k,-m,-g 规定显示单位
    • -h 可以以不同的单位呈现
    • 格式：
      • free
      • free –h
  • top
    • 实时显示linux的进程
    • 大致了解下：
    • up:系统启动的时间
    • cpu：us：使用率，sy：内核占用cpu使用率，id：cpu闲置率
    • %CPU、%MEM：对应进程的资源占用
