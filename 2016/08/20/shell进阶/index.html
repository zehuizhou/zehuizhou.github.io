shell进阶

发表于 2016-08-20 | 分类于 shell | 阅读次数

shell说白了就是一堆命令的集合
但是这不是毫无目的的，需要有一定的语言逻辑
基本的包括：赋值语句，基本输入输出，条件判断，循环控制

输出命令

  • echo
    • 回显(打印)一个字符到终端
    • -n 不打印换行
    • echo “hello world!”
    • echo –n “abc”

重定向

  • >和>>
    • 重定向，讲一个命令的输出引导到文件里面通常我们使用>或者>>
    • > 如果文件不存在，则创建文件，如果文件存在，则清空内容
    • >> 如果文件不存在，则创建文件，如果文件存在，则追加内容
    • 扩展：
      • 2>和2>>表示的标准错误输出的流重定向
      • 2>&1 将标准错误输出重定向到标准输出

管道命令

  • |
    • 就是一个竖线，按住shift的同时，按Enter键上面那个键
    • 作用是引输出(流)为下一个命令的输入(流)
    • 举例子：
      • ls | more 就是将ls的结果用more的形式去查看，用在ls的结果超出一屏的时候
    • 后面会有很多用到|的地方只要理解下管道就是将上一个命令的标准输出引导到下个命令做标准输入，但前提是后面的命令需要可以接受标准输入。
    • 一个错误的例子：
      • cat a | ls ls是不支持标准输入的命令
      • 上面的命令篇的基本上都不支持标准输入，有兴趣的同学可以试试
      • 下面学习的命令只是都支持标准输入，并不用都藏在|之后

排序命令

  • sort
    • 我们处理问题基本离不开排序，shell提供了强大的排序功能sort
    • -d 按字典序排
    • -n 按数字序排
    • -r 反序排
    • -k 指定用来排序的行
    • -u 排序完成之后，删除重复的项
    • -t 指定分割符
    • 格式：
    • cat /etc/passwd | sort –t: -n –k3
    • cat /etc/passwd | sort

筛选命令

  • grep
    • 根据关键字来对输入流进行筛选
    • -i 不区分大小写
    • -v 排除某个关键字
    • -E 使用扩展正则表达式进行筛选 相当于用egrep
    • 格式：
    • cat /etc/passwd | grep –i bash | grep –v root筛选出包含bash但是不包含root的行
  • uniq
    • 删除重复的行
    • -c 统计行数
    • -i 忽略大小写
    • 和sort –u 对比的话，uniq只删除的是连续的重复行
    • 格式
      • cat file.txt| sort | uniq –ic
  • head tail
    • head 取头，tail 取尾
    • -n head取前几行 tail取后几行
    • 格式：
      • cat a.txt| head –n 10 | tail –n 3
      • 取文件a.txt的第8、9、10行
    • tail有一个监视的选项可以注意下
      • tail –f /var/log/dmesg
      • 实时监测dmesg的变化

字节流

  • wc
    • word count 字符计数器
    • -c 打印字节数
    • -m 打印字符数
    • -l 打印行数
    • -w 打印字数
    • 格式：
      • cat a.txt| wc –l
      • 统计a.txt的行数
  • tr
    • 替换或者删除字符
    • -d 删除
    • 格式
      • echo “hello world” | tr “[a-z]” “[A-Z]”
      • 将小写转化为大写 后面跟着两个字符集合 ASCII编码
      • echo “ hello world ”| tr –d “[ ]”
      • 删除所有的空格
    • tr可以用来将windows下面的文件转化为linux的文件
      • 使用tr –d “[\r]” 删除回车保留换行
  • sed
    • stream editor 流编辑器
    • 功能太太太太强大，只打算讲替换功能
    • -n 不回显整个输入流
    • -e 多个联合
    • p 打印
    • s 替换
    • g 全局
    • 格式：
      • sed –n ‘8,10p’a.txt
      • 打印a.txt的第8,10行
      • sed –e ‘s/aaa/bbb/g’ a.txt
      • 将a.txt的aaa全部替换成bbb

文本块

  • awk
    • 文本分析工具
    • 功能也是太太太太强大，将用的最多的取字段
    • -F 指定分隔符
    • $n 取第n列的数据，$1为第一列 $2为第二列 $NF为最后一列
      • NF是awk的内置变量，表示最后一个域的标号
    • awk -F: ‘{print $NF}’ /etc/passwd
      • 打印出passwd以:为分隔符的最后一列
    • awk ‘/abcde/ {print}’ a.txt
      • 打印出包含abcde的行
    • awk ‘{if (NR >= 8 && NR <= 10) print}’ a.txt
      • 打印a.txt的第8,9,10行 NR是awk的内置变量，存储着当前处理的行号

学习了一些命令，接下来学习语句
脚本的第一行都是注释语句，声明本脚本使用的是哪一种的shell
我们平时接触的只有两种，bash和sh，所以我们声明注释的时候一般是#/bin/bash或者#/bin/sh
注释以#开头 之后的脚本在执行的时候就会自己跳过了

赋值

  • =
    • 就是赋值一个变量，存储一个数据
      • var=aaa
      • echo $var
    • ``和$()这两者是等价的，为取一个命令的基本输出
      • var1=`ls`
      • ls是用``
      • var2=$(ls)
      • echo $var1 $var2
        *注意：赋值的时候=两边不要有空格！
    • 一些特殊的变量
      • $1 传入的第一个参数 $? 上一个命令执行的返回码 $$当前的进程号
      • $@/$* 传入的参数组
      • $@所有参数为单个 “$1” “$2” “$3” 这种 $*所有参数作为整体”$1 $2 $3”这种
      • $# 传入的参数的总个数

输入输出

  • echo read

    • echo 是输出，还有个输出是printf，这个可以用来格式化输出，一般来讲有特殊的需求才会用到，用法是和C语言一样

    • read 读取用户的输入或者是读取文件，数据都是以行为单位存储的

    • 例子：

      • read –p “tips” var
      • 打印出“tips”，等待用户输入
      • read –n 3 var
      • 最多3个字符输入给变量var
      • read –t 30 var
      • 等待30秒，如果用户没有输入，则退出，var变量的值不变

条件判断语句

  • if
    • [ ]为取一个表达式的布尔结果，在’[‘之后要有空格，在’]’之前要有空格
    • && 为与 ||为或 !为非
    • -eq 相等 –ne 不等 –lt 小于 –le 小于等于 –gt 大于 –ge 大于等于
    • -e 存在 –n 非空 –z 空
    • == 判断字符串相等
    • 格式 
       
if [ “$a” –eq 1 ] || [ “$b” == “miss” ]  
then  
echo “hide”  
elif [ -e “/etc/file” ] && [ ! –z “$d” ]  
then  
echo “SHE”  
else  
echo “n”  
fi
  • case
    • 相当于很多个条件判断，和c语言里面的switch是一个
    • 格式：

      case $var in:
      1)  echo j ;;
      2)  echo 0 ;;
      3)  echo 11 ;;
      4)  echo y ;;
      *)  echo one ;;
      esac

      当var为1的时候输出j,以此类推，当不是1234的时候，输出one！

循环

while和for是循环的两个关键字，例如我们要输出0-100，有如下的方式：

i=0  
while $i <= 100  
do  
echo $i  
i=$(($i+1))  
done  

for((i=0;$i<=100;i++))  
do  
echo $i  
done  

for i in $(seq 0 100)  
do  
echo $i  
done
